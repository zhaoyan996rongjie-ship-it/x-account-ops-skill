# Engagement Templates

## Reply Patterns by Scenario

### 1. Tool Comparison / Token Cost Posts
When someone discusses AI tool pricing or token usage:
```
"This is exactly what I've been tracking! I ran a comparison between [Tool A] and [Tool B] and found [Tool B] consumes ~2-3x more tokens for similar tasks. Would be great to compare notes on cost optimization strategies."
```

### 2. Someone New to an AI Tool
When someone just started using a tool:
```
"Welcome to [Tool]! 🎉 I've been tracking my usage closely — for [specific task type], [Tool] is surprisingly efficient compared to other options. Would love to hear how your first week goes. What kind of projects are you building?"
```

### 3. Tips & Prompt Engineering
When someone shares prompt tips:
```
"Great tips! I've found that [specific technique] saves a lot of token waste. Small changes in prompting really do make a big difference in both output quality AND cost. Have you tried [related technique]?"
```

### 4. AI-Generated Code Quality
When someone discusses AI code quality:
```
"This is a real concern. In my experience, AI code quality varies significantly by tool and use case. The key is knowing where AI excels (boilerplate, tests) vs where human review is critical (security, architecture)."
```

### 5. Philosophical/AI Future Posts
When someone discusses AI's future:
```
"Great perspective! I think the real shift isn't just about better prompts — it's about understanding which workflow fits which tool. The token efficiency difference between tools is a practical concern that doesn't get enough attention."
```

## Engagement Do's and Don'ts

### Do:
- Add new information or perspective
- Reference specific experiences
- Ask open-ended questions
- Keep replies 1-3 sentences
- Match the tone of the original post

### Don't:
- Just say "Great post!" or "Nice!"
- Copy-paste the same reply to multiple posts
- Self-promote unless directly relevant
- Argue or correct aggressively
- Engage with obvious spam/crypto posts
